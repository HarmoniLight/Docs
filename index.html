MicroBit Lighting Communication Standards

Owen Plimer - Jan 2025

Project Description

This manual describes the communication standard to be used for the 2025 Software Development with Data Analytics HND Graduates ceremony light display. There are 2 different phases of this project from a technical standpoint. Phase 1 (Alpha) can only be used to change the brightness of the on-board LED display on the MicroBits. Phase 2 (Beta) will be full RGB LED control capabilities. Phase 1 must be completed before phase 2 can begin.

Definitions

Definition Description
Controller This is the MicroBit which is connected to the laptop/PC of the perosn who is running/designing the light display
Reciever This is the MicroBit that is being worn by the student

Alpha

Controller

When the controller is powered on, it will wait for instruction from the controlling PC. The controller is responsible for converting the serial commands to radio signals and also the opposite direction (Radio -> Serial) There are 3 operation modes for the controller.

  1. Power-On State (POS)

  2. Device Discovery Mode (DDM)

  3. Command Mode (CM)

    POS

    The initial state when the device powers on. In this state, the device is listening for the byte string confirm. This string tells the controller that the desktop app is running and ready to connect to it. When the controller recives this command, it responds with the byte string confirmed. This lets the ShowRunner know that a controller is connected and ready to recieve commands.

CM

After discovery is complete, the controller enters CM. This is where it acts as a Serial -> Radio converter for the ShowRunner to send commands to the Recievers. The controller takes an arrayIndex for the reciever being targetted, and a target brightness and sends it over radio as follows:  arrayIndex = targetBrightness. When it recieves this string, the controller will send it over the radio frequency for the show.

Reciever

The recievers have 2 modes of operation: 4. Pre-Discovery (PrD) 5. Post-Discovery (PoD)

PrD

This is the default mode at startup. It is enabled when the discovered variable is set with the value false. In this mode, when the controller sends the discover command, the reciever unit waits for a random number of seconds and replies with the me command in the format me = SerialNumber where SerialNumber is the unique serial number of the reciever MicroBit. It then waits for the controller to reply with the ID of the unit. The ID of the reciever is generated by the Controller and it is the index in the Recievers array on the controller. The reciever then sets the ID variable to the ID returned by the controller. It then proceeds to Mode 1 (PoD)

PoD

This mode is triggered when the discovered variable is set to true, and when the ID variable has a value other than None. In this mode, the Reciever listens for new target brightnesses and actions them. The Psuedocode for the Listener and Actioner are shown below.

// Actioner Script
SET targetBrightness = 0
SET currentBrightness = 0
Forever Loop:
    // Runs forever in main process
    If targetBrightness < currentBrightness:
        // Going down to target brightness
        currentBrightness--
        setBrightness(currentBrightness)
    If targetBrightness > currentBrightness:
        // Going up to target brightness
        currentBrightness++
        setBrightness(currentBrightness)
    Else:
        // We're at the target brightness, do nothing
    
    pause(100) // In MilliSeconds
// Listener Script
RadioListener:
    SET recievedName = "" // This comes from the radio recieved event OOS
    SET recievedValue = 0 // This comes from the radio recieved event OOS
    
    SET targetID = recievedName.toInt()
    SET targetBrightnessValue = recievedValue
    If targetID = Me.ID:
        // I am the reciever being spoken to
        If targetBrightnessValue = Me.targetBrightness:
            // Go straight to the target.
            Me.currentBrightness = targetBrightnessValue
            setBrightness(Me.currentBrightness)   
        Else:   
            Me.targetBrightness = targetBrightnessValue
    

The reciever can be instructed to do two different things in this mode. 6. Fade to a target brightness 7. Go straight to a target brightness

Fade To Brightness

To fade to a brightness, the command should be sent once to the reciever. This will allow the default Actioner Script to run through the sequence to quickly fade to the target brightness.

Go Straight to Brightness

To go straight to a brightness. The value must be sent to the reciever twice in quick succession. This will override the fade sequence.
